Offside canvas test HTML

Depending on our needs some part of the CSS may be modified (see HTML). f.e: we can change the size of the menu panel, or the space we keep from the main content after menu slides in.

By default there's also an exit panel class defined. With z-index it's aligned so, that it is always above the main content but below the menu. Exitting the menu can be achieved also by adding a button (or anchor tag) with the proper data- elements - see exit link.

NOTE:Offside.js and offside.less (css) (and jquery) must accessible for this example to run.

Multi-panel menu

Currently we have configured 5 panels in the CSS. I kept every important detail (as description) so if it's needed it's easy to modify CSS and support any 1..N panels. The javascript code must not be modified to achieve this, it's enough to change/extend the CSS.

Separate exit button can be configured (the same way as with slide/push in menus) - but what it only does is switching to the default panel. And since I assume there will be always a visible buttom to change panels from 1..N, the exit has no real meaning in this case.
